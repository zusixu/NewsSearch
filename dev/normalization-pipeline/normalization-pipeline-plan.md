# normalization-pipeline

## 做什么

将采集到的原始文档去重、清洗、标准化，并转换为可供事件抽取和链路构建使用的中间结构。

## 怎么做

1. 设计 `RawDocument -> NewsItem -> EventDraft` 转换流程。
2. 实现 URL 指纹、文本哈希去重。
3. 标准化时间、来源、标题、正文。
4. 增加来源可信度分级。
5. 输出可复用的标准化数据对象。

## 完成定义

- 同类重复新闻可被稳定合并。
- 时间和来源字段格式一致。
- 事件抽取层可直接使用标准化结果。
